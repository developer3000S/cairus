    Анализ команд TUI CAI: команды Agent, Model и Parallel

    Обзор

    Этот документ анализирует работу команд CAI вроде /agent, /model и /parallel в контексте TUI, сосредотачиваясь на различии между глобальным состоянием и состоянием для каждого терминала.

    Архитектура

    Основные компоненты

    1. CommandHandler (src/cai/tui/components/command_handler.py)
    2. Обрабатывает выполнение CLI-команд внутри TUI
    3. Перехватывает вывод консоли и перенаправляет в подходящие виджеты терминала

    4. Имеет специальную обработку для команд /agent и /model

    5. SessionManager (src/cai/tui/core/session_manager.py)

    6. Управляет общим состоянием сессии TUI
    7. Координирует несколько терминальных исполнителей
    8. Управляет координацией параллельного режима

    9. Методы: update_model(), switch_agent()

    10. TerminalRunner (src/cai/tui/core/terminal_runner.py)

    11. Управляет выполнением агента внутри одного терминала
    12. У каждого терминала есть свой экземпляр агента и история сообщений

    13. Методы: switch_agent(), update_model()

    14. REPL команды (src/cai/repl/commands/)

    15. /agent — выбор и управление агентом
    16. /model — выбор модели
    17. /parallel — настройка параллельного агента

    Анализ поведения команд

    1. Команда /agent

    Текущая реализация: - Изменяет глобальную переменную окружения CAI_AGENT_TYPE - Обновляет глобальное состояние AGENT_MANAGER - Обрабатывает загрузку параллельных шаблонов в PARALLEL_CONFIGS

    Проблемы в контексте TUI: - Команда изменяет глобальное состояние, влияющее на все терминалы - Нет механизма выбора агента для каждого терминала - CommandHandler имеет специальную обработку, но обновляет только локальное состояние

    Требования для каждого терминала: - Нужно вызвать session_manager.switch_agent(agent_name, terminal_number) - Не должна изменять глобальную переменную CAI_AGENT_TYPE в режиме TUI - Каждый терминал должен иметь собственную конфигурацию агента

    2. Команда /model

    Текущая реализация: - Изменяет глобальную переменную окружения CAI_MODEL - Влияет на все будущие взаимодействия агентов глобально

    Поведение в контексте TUI: - CommandHandler обнаруживает команды /model - Вызывает session_manager.update_model(model_name) - SessionManager обновляет всех терминальных исполнителей с новой моделью - Это подходящее поведение для изменения модели (обычно требуется единая модель для всех терминалов)

    Работает корректно: - Обновления модели распространяются на все терминалы как ожидается - Каждый терминальный исполнитель рекурсивно обновляет модель своего агента

    3. Команда /parallel

    Текущая реализация: - Управляет глобальным списком PARALLEL_CONFIGS - Устанавливает переменные окружения CAI_PARALLEL и CAI_PARALLEL_AGENTS - Настраивает агентов для параллельного выполнения

    Поведение в контексте TUI: - SessionManager обнаруживает параллельный режим через флаг is_parallel_mode - Распределяет параллельных агентов между терминалами (Терминал 1 = P1, Терминал 2 = P2 и т.д.) - Использует PARALLEL_CONFIGS для определения назначений агентов

    Проблемы: - Параллельный режим глобален, не специфичен для терминала - Невозможно иметь разные конфигурации параллельных агентов в разных терминалах - Команда /agent, загружающая параллельные шаблоны, влияет на все терминалы

    Управление состоянием

    Глобальное состояние (общее для всех терминалов)

    • Переменные окружения: CAI_AGENT_TYPE, CAI_MODEL, CAI_PARALLEL
    • Список PARALLEL_CONFIGS
    • Реестр AGENT_MANAGER
    • Истории PARALLEL_ISOLATION

    Состояние для каждого терминала

    • TerminalRunner.agent — экземпляр агента
    • TerminalRunner.message_history — история беседы
    • TerminalRunner.config — конфигурация терминала, включая agent_name и model
    • TerminalConfig.parallel_config — назначение параллельного агента

    Рекомендуемые исправления

    1. Сделать /agent терминально-зависимой в режиме TUI

    # В command_handler.py, улучшите специальную обработку:
if cmd_name.lower() in ["/agent", "/a"] and args:
    if args[0] == "select" and len(args) > 1:
        agent_name = args[1]
        terminal_number = self.terminal_number  # Требуется добавить

        # Не изменяйте глобальное состояние в режиме TUI
        if hasattr(self, 'session_manager') and self.session_manager:
            asyncio.create_task(
                self.session_manager.switch_agent(agent_name, terminal_number)
            )

    2. Добавить команду для конкретного терминала

    Создайте новую команду типа /terminal-agent или изменить /agent для принятия номера терминала: - /agent select <name> --terminal 2 — выбрать агента для конкретного терминала - /agent select <name> — в TUI влияет только на текущий терминал

    3. Расширить параллельный режим для TUI

    • Разрешить конфигурации параллельных агентов быть специфичными для набора терминалов
    • Каждый терминал может иметь собственную группу параллельных агентов
    • Терминалы 1-3 могут запускать один параллельный набор, терминалы 4-6 другой

    4. Добавить обнаружение контекста TUI

    Команды должны обнаруживать режим TUI и вести себя иначе:

    # Текущее поведение — изменить глобальное состояние
    os.environ["CAI_AGENT_TYPE"] = agent_key
else:
    # Поведение TUI — уведомить session manager
    # Не изменяйте глобальное состояние

    Текущие обходные решения

    1. Для выбора агента в конкретном терминале:
    2. Используйте прямое выполнение в терминале вместо команд

    3. Вручную переинициализируйте терминальные исполнители после глобальных изменений

    4. Для изменения модели:

    5. Текущая реализация хорошо работает для глобальных обновлений модели

    6. Используйте команду /model нормально

    7. Для параллельного режима:

    8. Настройте параллельных агентов перед запуском терминалов
    9. Используйте переменные окружения для предварительной настройки
    10. Параллельный режим влияет на все терминалы по назначению

    Заключение

    Система команд TUI в настоящее время сильно зависит от глобального состояния, унаследованного из дизайна CLI. Для поддержки настоящих многотерминальных рабочих процессов команды должны быть расширены с поддержкой терминально-зависимых операций и способностью изменять состояние для каждого терминала, а не глобальное состояние. Команда /model хорошо работает в своём нынешнем виде, поскольку изменения модели обычно желательны глобально, но /agent и /parallel нуждаются в реализации для конкретного терминала.
